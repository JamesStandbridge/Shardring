# Systeme Economy

## Role

L'economie cree des decisions de risque/recompense pendant la run et prepare une progression long terme plus rare.

## Contrat futur

- Monnaie de run : apparait pendant une partie, se depense via une mecanique d'achat future, se remet a zero a la mort.
- Monnaie cross-game : rare, persistante, attribuee plus tard selon les resultats de run.
- La mecanique d'achat ne doit pas supposer un shop central permanent.
- Les achats doivent refuser les depenses impossibles et appliquer les upgrades via un contrat clair.
- Les valeurs de prix et rarete doivent etre configurees par donnees.

## Limites actuelles

La monnaie cross-game doit etre documentee et reservee dans l'architecture, mais non implementee avant stabilisation de la boucle de run.
